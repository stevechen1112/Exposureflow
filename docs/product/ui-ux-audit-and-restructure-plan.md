## 五、重構方案（大刀闊斧）

### 5.1 導航重構（P0）

**現狀**：14 項站點分析平鋪 + 7 項工作區平鋪 = 21 項無階層

**方案**：改為 **6 個語意群組**：

```
📊 總覽
  ├─ 曝光儀表板          /dashboard        ← 重構為 Action Dashboard
  └─ 多站總覽            /agency           ← 修復 B2

🎯 策略與研究
  ├─ 策略 Intake          /strategy         ← 視覺化（5.11）
  ├─ 關鍵字金字塔         /keyword-pyramid  ← 拆分 3 Tab（5.2）
  └─ SERP 版位矩陣        /serp-matrix

📋 機會與決策
  ├─ 機會佇列            /opportunities    ← 加 empty CTA
  ├─ 執行路線圖           /roadmap
  └─ 交付承諾            /delivery-commitments

✍️ 內容與發布
  ├─ 內容審核            /content-review   ← 修復 B3 + 工作流重設計（5.10）
  ├─ 知識庫              /knowledge
  └─ AI 與品牌能見度      /ai-brand         ← 合併 AI 能見度 + SERPO + 品牌實體（5.12）

📈 量測與診斷
  ├─ 曝光地圖            /exposure-map     ← 修復 B1
  ├─ 技術問題            /technical-issues ← 修復 B1
  └─ 行動成果            /outcomes         ← 修復 B4

⚙️ 工作區設定
  ├─ 新站上線            /onboarding
  ├─ 站點                /settings/sites
  ├─ 設定                /settings
  ├─ GSC 連線            /settings/integrations
  ├─ 成員                /settings/members
  └─ 計費                /settings/billing
```

導航項從 21 → **17**（合併 3 頁為 1，移除重複）。

### 5.8 儀表板重構：從「資料展示」轉為「行動導向」（P0）

**現狀**：純資料展示，看完不知下一步。

**方案**：重構為 **Action Dashboard**——

- 頂部增加 **「需要你的注意力」區塊**（從各模組彙總 actionable items，每項可點擊直達）
- KPI 卡從 6 個減為 3 個核心（曝光、查詢覆蓋、已索引）
- Topic Cluster 表每列可點擊 → 跳轉曝光地圖並自動篩選
- 排名分佈區塊可點擊展開查詢列表

### 5.9 跨頁連結與工作流（P0）

建立 SEO 專員的自然操作路徑：

```
策略 Intake → 關鍵字金字塔 → SERP 版位矩陣 → 曝光地圖（看 gap）
    │  點擊「補缺口」
    ▼
內容審核（建立 Brief → Pipeline 生成 → 審核 → 發布）
    │
    ▼
行動成果（追蹤成效）→ 回饋到曝光地圖（gap → covered）
```

- 曝光地圖「補缺口」→ 直達內容審核，自動帶入關鍵字 + action_type
- 機會佇列「核准」→ 自動建立 Roadmap item
- Pipeline 完成 → 自動更新 Roadmap 狀態

### 5.10 內容審核工作流重設計（P1）

重設計為 **4 步驟線性工作流**（向 ContentFlow 看齊）：

- Step 1：選擇曝光機會（修復後的下拉）
- Step 2：建立 Source Pack → Content Brief
- Step 3：AI 生成 + SEO Check（Pipeline，進度指示器）
- Step 4：人工審核 → 核准 → 發布

步驟之間有視覺連接，每步完成後自動展開下一步。

## 七、重構原則

1. **不影響後端 API**：所有重構限於前端，後端路由/邏輯保持不變
2. **不破壞現有功能**：評分面板、Pipeline、SERP 富化等已交付功能完整保留
3. **漸進式**：P0 → P1 → P2，每階段可獨立 deploy
4. **以 Onboarding 頁為品質標竿**：8 步驟 + 策略對照 + CTA
5. **大刀闊斧是允許的**：導航結構、頁面拆分、術語命名都可大幅改動
